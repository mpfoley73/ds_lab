Goal

Build Up a Table

Let’s build up a {gt} table from scratch. Here is a reference schematic from the gt website:

The first thing to notice is that the object has a table body and column labels. It can potentially have a stub (row groups or labels), header, and footer.

This is analogous to the {ggplot} object. “gt” stands for “grammar of tables”, and just like {ggplot} layers build from a base, {gt} functions pipe together to add structure and styling.

1. The Basic Table

gt() converts a data frame into a gt_tbl object. That’s all you need, but you can optionally add tab_header(), cols_label(), tab_stubhead(), and tab_sourcenote().

Show the code 
tbl_1 <- gt_cars_sample |>
  gt::gt(rowname_col = "mfr") |>
  gt::tab_header(
    title = "Basic Table",
    subtitle = "with stub, column labels, header, and footer."
  ) |>
  gt::cols_label(
    model = "Model",
    year = "Year",
    hp = "Horsepower",
    mpg_c = "MPG - City",
    mpg_h = "MPG - Hwy"
  ) |>
  gt::tab_stubhead("Manufacturer") |>
  gt::tab_source_note("Source: gtcars is from the gt package.")

2. Footnotes

You can add footnotes almost anywhere on the table. Base command is tab_footnote(footnote, locations) where locations is a function related to a part of the table:

  • cells_body()
  • cells_column_labels()
  • cells_column_spanners()
  • cells_grand_summary()
  • cells_group()
  • cells_row_groups()
  • cells_source_notes()
  • cells_stub()
  • cells_stub_grand_summary()
  • cells_stub_summary()
  • cells_stubhead()
  • cells_title()
Show the code 
tbl_2 <- tbl_1 |>
  gt::tab_footnote(
    "gt Cars is gtcars in gt.",
    locations = gt::cells_title(groups = "title")
  ) |>
  gt::tab_footnote(
    "Horsepower over 600 is a strong vehicle.",
    locations = gt::cells_column_labels(columns = hp)
  ) |>
  # Footnote in table body. `rows` uses a formula.
  gt::tab_footnote(
    "Fuel efficient!",
    locations = gt::cells_body(
      columns = mpg_h,
      rows = mpg_h > 20
    ),
    placement = "right" # puts the marker on the right side of the cell
  ) |>
  gt::tab_header(
    title = "Footnotes",
    subtitle = gt::md(
      "In header (`cells_title()`), <br>column labels (`cells_column_labels()`), and <br>body(`cells_body()`)"
    )
  )

3. Formatting

tab_options() is the {gt} equivalent of ggplot2::theme(). To format specific cells, use tab_tyle(style, locations) where style is a function related to a styling type:

  • cell_text()
  • cell_fill()
  • cell_borders()
Show the code 
tbl_3 <- tbl_2 |>
  # TABLE DEFAULTS
  gt::tab_options(
    heading.align = "left",
    table.font.size = gt::pct(90)
  ) |>
  # SPECIFIC CELLS
  gt::tab_style(
    style = gt::cell_fill("gray90", alpha = .5),
    locations = gt::cells_column_labels(
      columns = c(model, year, hp, mpg_c, mpg_h)
    )
  ) |>
  gt::tab_header(
    title = "Formatting",
    subtitle = gt::md("with `tab_options()` and `tab_style()`.")
  )

Takeaways

You now have the mental model and toolkit to create polished tables for 80% of real-world situations. We covered:

  • Structure: Use stubs, row groups, and column spanners to organize data logically.
  • Context. Use headers, footers, and footnotes to explain what readers are looking at.
  • Style. Use colors, text formatting, and heat maps to draw attention to what matters.

From here:

  • The {gt} documentation is comprehensive and you know the conceptual landscape well enough to navigate it.
  • Explore the ecosystem: {gtSummary} for statistical summaries, *{gtExtras}** for plots and spark lines.

## Build Up a Table

Let's build up a **{gt}** table from scratch. Here is a reference schematic from the [gt](https://gt.rstudio.com/) website:

![](gt_parts_of_a_table.svg)

The first thing to notice is that the object has a *table body* and *column labels*. It can potentially have a *stub* (row groups or labels), header, and footer.

### 1. The Basic Table

`gt()` converts a data frame into a `gt_tbl` object. That's all you need, but you can optionally add `tab_header()`, `cols_label()`, `tab_stubhead()`, and `tab_sourcenote()`.

#| code-fold: true

    subtitle = "with stub, column labels, header, and footer."
  ) |>

- `cells_row_groups()`
